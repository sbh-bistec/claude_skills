BMAD Beginner's Guide

What is BMAD?

BMAD is an open-source framework that supercharges Agile software development by embedding specialized AI agents into every phase — from ideation to delivery.

Traditional Agile + AI copilots = faster decisions, better specs, less rework.

Agile-native

Built around sprints, backlogs & user stories — just AI-powered.

Agent-driven

Each role (PM, Architect, Dev) has a dedicated AI persona.

Open source

Community-maintained prompts, workflows & templates on GitHub.

BMAD Beginner's Guide

Phase 1: Discovery

Agent: Analyst
Goal: Understand what we are building and why.
  • Analyst agent interviews you with structured questions
  • Captures business goals, target users, constraints
  • Produces a Project Brief document
  • Human reviews and approves before moving on
Output file: docs/project-brief.md
BMAD Beginner's Guide

Phase 2: Planning

Agent: Product Manager
Goal: Convert the brief into an actionable product roadmap.
  • PM agent reads the Project Brief
  • Generates a full PRD with epics & user stories
  • Each story has clear acceptance criteria & priority
  • Backlog is sized and sprint-ready

Epic

Large feature area (e.g. "User Auth System")

Story

"As a user, I can reset my password via email."

Criteria

Testable conditions that define "done".

BMAD Beginner's Guide

Phase 3: Architecture

Agent: Architect
Goal: Design the technical foundation before writing code.
  • Architect agent reads PRD + constraints
  • Proposes tech stack, folder structure, data models
  • Defines API contracts and integration points
  • Creates Architecture Doc + coding standards
  • Human approves the design — locking the foundation
Output files: docs/architecture.md, docs/tech-stack.md
BMAD Beginner's Guide

Phase 4: Development

Agent: Developer
Goal: Implement stories one at a time, guided by docs.
  • Dev agent loads the Architecture Doc + target story
  • Writes code following established patterns
  • Story is marked done only when all criteria pass
  • Human reviews each story before the next begins

Dev works on:

One story at a time. No scope creep.

You review:

Code diff, tests pass, acceptance met → merge.

Phases 5 & 6: QA & Deploy

Agent: QA Engineer
  • Reviews code for quality issues
  • Writes / runs automated tests
  • Produces a QA Report
  • Flags regressions before release
Agent: Orchestrator
  • Confirms all stories complete
  • Reviews QA sign-off
  • Guides deployment checklist
  • Updates project status docs
The human always pulls the final deploy trigger. BMAD ensures everything is ready before you do.

Getting Started with BMAD

1

Install the framework

Clone the BMAD repo or install the Claude Code extension. Copy agent prompts into your project.

2

Create your docs/ folder

This becomes the shared memory. Start with project-brief.md.

3

Activate the Analyst agent

Describe your idea. Let the agent ask clarifying questions.

4

Follow the workflow

Discovery → Planning → Architecture → Dev → QA → Deploy.

Tips for Beginners

  • Always read agent outputs before approving — AI is fast but not infallible.
  • Keep your docs/ folder updated. It's the agents' brain.
  • One story per session. Don't rush multiple stories at once.
  • If output feels off, refine the brief or add context — not more prompts.
  • Use the Orchestrator to check overall project health regularly.
  • Start with a small side project to learn the rhythm before using on critical work.
